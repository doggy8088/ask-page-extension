# 變更日誌

所有重要變更都會記錄在此檔案中。

格式基於 [Keep a Changelog](https://keepachangelog.com/zh-TW/1.0.0/)，並且本專案遵循 [語意化版本](https://semver.org/lang/zh-TW/)。

## [0.35.2] - 2026-07-22

### 新增 / 改進（v0.35.2）

- **Gemini 模型更新**：新增 `gemini-3.6-flash` 與 `gemini-3.5-flash-lite` 模型，並調整模型排序讓最新模型優先顯示於選項前方。

## [0.35.0] - 2026-07-20

### 新增 / 改進（v0.35.0）

- **完整保留詢問上下文**：詢問模式改用 DOM、ARIA 屬性與可見文字建立近似無障礙樹，保留互動角色、名稱、狀態、連結及表單欄位資訊；密碼欄位不會送出密碼值，建立失敗時才降級使用頁面文字。
- **固定頁面快照與完整對話**：第一問會固定頁面內容、選取文字與系統提示，後續追問完整保留文字、圖片及多輪對話，不再自動裁切或摘要；執行 `/clear` 或重新整理頁面時才重置。
- **提示詞快取與 Token 計量**：依提供者套用原生提示詞快取策略，並補充快取讀取與寫入 Token 統計，降低重複上下文的處理成本。
- **Gemini 預設使用 Google Search**：所有 Gemini 模型在詢問模式預設啟用 `google_search`；Gemini 3 代理模式可同時使用 Google Search 與頁面函式工具，其他模型則保留相容的頁面工具行為。
- **Gemini API 開發技能**：新增 Gemini API 模型、SDK 與官方文件查證規範，協助後續維護時使用目前支援的介面與模型資訊。

## [0.34.0] - 2026-07-13

### 新增 / 改進（v0.34.0）

- **新增 Ollama Cloud 提供者**：使用 `https://ollama.com/v1` 與 Bearer API Key 呼叫 Ollama Cloud，設定頁內建 6 個精選 Cloud 模型 ID 的 Checkbox 複選清單，並可從 `/v1/models` 載入最新完整清單。對話請求改由擴充功能 Service Worker 發送並轉送串流回應，避免受到目前網頁來源的 CORS 原則阻擋。

## [0.33.0] - 2026-07-11

### 新增 / 改進（v0.33.0）

- **新增 GPT-5.6 模型選項**：OpenAI 提供者新增 `gpt-5.6-sol`、`gpt-5.6-terra` 與 `gpt-5.6-luna`；OpenRouter 提供者新增 GPT-5.6 標準與 Pro 模型 ID。
- **GPT-5.6 Responses API 相容性防護**：當 GPT-5.6 搭配 function tools 時意外進入 Chat Completions 路徑，現在會明確阻擋請求並提示必須使用 Responses API。
- **OpenRouter 工具呼叫錯誤處理**：GPT-5.6 不再因端點工具不相容而靜默退回一般文字模式，避免在未提示的情況下遺失頁面操作能力。
- **保留既有 GPT-5 行為**：正常 GPT-5.6 Responses API 請求維持既有 `medium` 推理設定、工具呼叫、多輪對話與輸出處理流程。

## [0.32.2] - 2026-07-11

### 修正（v0.32.2）

- **修正自訂命令截圖設定 Checkbox 閃爍與尺寸突變問題**：在偏好設定頁面中，將全域 `input` CSS 樣式排除 `checkbox` 與 `radio` 類型，防止在「新增自訂命令」時點擊「啟用截圖模式」或選取模式單選鈕時，元素因誤套用 padding、box-shadow 與 focus 樣式而造成尺寸突增與畫面閃爍。

## [0.32.1] - 2026-07-08

### 改進（v0.32.1）

- **自訂命令預覽單行顯示**：使用提示展開時，自訂命令的提示詞預覽由最多兩行縮短為一行（`-webkit-line-clamp: 2 → 1`），確保兩欄左右對齊排版整齊一致。
- **移除「查看更多」按鈕，改為標題計數連結**：「自訂命令」區塊底部的「查看更多（另有 N 個）」按鈕移除，改為在標題「自訂命令」後方附加可點擊的總數標記 `(N)`，點擊後同樣開啟偏好設定頁面，版面更簡潔。
- **隱藏使用提示複製按鈕**：「使用提示」說明區塊右上角的複製按鈕設為 `display: none`，因該區塊為說明性 UI，不需提供複製功能。

## [0.31.0] - 2026-07-06

### 新增 / 改進（v0.31.0）

- **新增 Mistral AI 提供者**：提供者類型新增 Mistral AI，支援預設模型清單與 API 連線測試，並且能動態載入 Mistral 所有可用模型。預設模型選項包含 `mistral-large-latest`、`mistral-medium-latest`、`mistral-small-latest`、`codestral-latest`、`devstral-latest`。
- **OpenAI Compatible 複選模型**：點擊「🔍 載入端點模型」按鈕之後，提供完整的 Checkbox 讓使用者勾選，支援複選模型，與其他 AI 提供者一致。
- **動態模型輸入框**：在按下「🔍 載入端點模型」之前顯示手動輸入模型 ID 的文字框，載入模型後自動隱藏，避免重複輸入。支援使用英文或中文逗號分隔多個手動輸入的模型名稱。
- **動態切換手動輸入**：載入模型後，按鈕會變更為「✏️ 手動輸入模型名稱」，點擊可隨時切換回文字輸入框模式，並自動將所有勾選的模型轉換為逗號分隔。
- **保留手動輸入模型**：執行「🔍 載入端點模型」時，若手動輸入的模型不在載入清單中，會自動併入核取清單中並預設選取，避免自訂模型 ID 遺失，並透過去重邏輯防止模型 ID 重複。
- **優化偏好設定介面版面**：自訂系統提示詞追加框高度放大兩倍以利編輯；「新增 AI 提供者」與「新增自訂斜線命令」按鈕分別從清單最下方移至標題列最右側，垂直置中排列，提升版面緊湊度與操作效率。
- **系統提示自動儲存**：移除偏好設定頁頂端已無實際作用的「💾 儲存」按鈕；自訂追加提示詞在輸入時會自動進行去抖動（debounce）儲存，無須手動存檔。
- **記憶偏好設定頁籤**：自動儲存使用者最後選取的偏好設定頁籤（AI 提供者、自訂斜線命令、調整系統提示），下次開啟偏好設定頁面時將會自動切換並顯示該頁籤。
- **修正編輯對話框按鈕消失問題**：修復 `settings.html` 中 Groq 區塊漏掉閉合 `</div>` 導致 Mistral/Ollama 與儲存/取消按鈕被錯誤嵌套隱藏的問題。
- **修正連線測試驗證**：OpenAI Compatible 連線測試時不再強制要求填寫模型名稱（因為測試端點僅需要 API Endpoint 與 API Key 即可）。

## [0.30.0] - 2026-07-03

## 版本說明

### 版本號格式

我們使用語意化版本號: `主版本.次版本.修訂版本`

- **主版本**: 不相容的 API 變更
- **次版本**: 新增功能但向後相容
- **修訂版本**: 向後相容的錯誤修正

### 變更類型

- **新增**: 新功能
- **改進**: 現有功能的改善
- **修正**: 錯誤修正
- **安全性**: 安全性相關的修正
- **移除**: 移除的功能
- **廢棄**: 即將移除的功能
